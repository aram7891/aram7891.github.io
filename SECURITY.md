# Security Policy

## Supported Versions

This project does not use semantic versioning and does not publish tagged releases.  
All updates, including security fixes, are applied directly to the `main` branch.

## Reporting a Vulnerability

If you need to report a security vulnerability, please use GitHub’s private Security Advisories for this repository.

Reports are reviewed within approximately 48 hours.  
If the issue is confirmed, a fix will be prepared and coordinated under responsible disclosure before any public announcement.
